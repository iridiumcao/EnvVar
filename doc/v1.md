# 项目名称：EnvVar（MVP V1）

## 1. 项目目标

开发一个 Windows 11 下的环境变量可视化管理工具，
提供基础 CRUD + 信息增强 + 多值变量解析能力，
提升开发者对环境变量的可读性与可操作性。

---

## 2. 功能范围（V1）

### 2.1 环境变量 CRUD

支持以下操作：

- 查看环境变量列表
- 新增环境变量
- 编辑环境变量（key / value）
- 删除环境变量

支持变量级别：

- User（用户级）
- System（系统级）

---

### 2.2 数据增强（本地扩展信息）

为每个环境变量提供附加信息（仅存本地）：

- alias（用户自定义名称）
- description（备注说明）

存储方式：
- 本地 JSON 文件

示例结构：

```json
{
  "JAVA_HOME": {
    "alias": "Java Home",
    "description": "JDK installation path"
  }
}
```

要求：
- 不影响系统真实环境变量
- key 作为唯一标识

---

### 2.3 展示模式

#### 模式一：分级展示

- 按 User / System 分组展示
- 每组内列表展示变量
- 基础排序（按 key）

---

#### 模式二：合并展示

- 所有变量合并为一个列表
- 每个变量标记 level（User / System）
- 使用视觉区分（颜色 / 标签）
- 基础排序（按 level, key, level + key）

---

### 2.4 多值变量拆分展示（重点）

适用于：
- PATH
- CLASSPATH
- 其他以分号（;）分隔的变量

功能：

- 自动识别 value 中的分号分隔结构
- 拆分为列表展示：

示例：

```plaintext
PATH =
C:\Java\bin;C:\Windows\System32
```

→ 展示为：

```plaintext
[0] C:\Java\bin
[1] C:\Windows\System32
```

---

展示要求：

- 每一项独立一行
- 显示顺序（索引）
- 保持原始顺序

---

编辑策略（V1 简化版）：

- 暂不支持单项编辑
- 仅支持整体编辑 value（字符串）
- 拆分展示仅用于“查看增强”

👉（这是关键取舍，避免复杂度爆炸）

---

### 2.5 基础交互

- 点击变量 → 查看详情（value + alias + description）
- 支持编辑 alias / description
- 支持刷新环境变量列表

---

## 3. 非功能需求

### 3.1 权限控制

- 修改 System 变量时提示管理员权限
- 操作失败需明确提示

---

### 3.2 数据安全

- 修改前读取最新系统变量
- 避免覆盖冲突

---

### 3.3 性能

- 启动时间 < 1 秒
- 支持变量数量：100+

---

### 3.4 可用性

- UI 简洁（表格 + 侧边详情）
- 操作路径清晰（避免隐藏操作）

---

## 4. 数据结构设计（建议）

### 4.1 系统变量结构（运行时）

```json
{
  key: string,
  value: string,
  level: "User" | "System"
}
```

---

### 4.2 扩展数据结构（本地 JSON）

```json
{
  [key: string]: {
    alias?: string,
    description?: string
  }
}
```

---

⚠️ 注意：

- 不建议直接用 key 作为唯一索引（因为 User/System 可能同名）
- 推荐使用：
  key + level 作为复合 key

示例：

```plaintext
"JAVA_HOME@User"
"JAVA_HOME@System"
```

---

## 5. 技术实现建议

### 5.1 系统读取

来源：

- User：
  HKCU\Environment

- System：
  HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Environment

---

### 5.2 更新机制

- 写入 Registry
- 通知系统刷新（广播 WM_SETTINGCHANGE）

---

### 5.3 多值解析

- 使用分隔符：
  ;

- 注意：
  - 去除空项
  - 保留原始顺序

---

## 6. 明确不做（V1）

以下功能**明确不进入 V1**：

- PATH 单项编辑（增删改排序）
- 快照 / 回滚
- 分组管理
- 搜索功能
- 导入 / 导出

---

## 7. 成功标准（V1 Done Definition）

满足以下条件即视为完成：

- 能完整展示系统 + 用户环境变量
- 能安全进行 CRUD 操作
- 能为变量添加 alias 和备注
- 能正确拆分并展示 PATH 类变量
- 无明显误操作风险（误删 / 覆盖）
