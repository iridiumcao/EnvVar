# EnvVar 日志系统需求文档

(The original version was generated by ChatGPT. I made some changes.)

## 1. 目标

为 EnvVar 提供一套简单、稳定、可维护的日志系统，用于：

* 记录应用运行过程中的关键行为
* 捕获和记录异常信息
* 支持用户反馈问题时提供日志文件

---

## 2. 日志设计原则

* 使用**统一日志系统**，不区分“用户日志”和“错误日志”
* 通过**日志级别（Log Level）**区分日志类型
* 保持实现简单，避免过度设计
* 优先保证**可读性和可排查性**

---

## 3. 日志级别（Log Levels）

支持以下级别：

* `Error`：异常和错误（必须记录）
* `Warning`：非致命问题
* `Information`：关键业务流程（默认级别）
* `Debug`：（可选，预留）

默认日志级别：`Information`

---

## 4. 日志存储

### 4.1 存储路径

```plaintext
%LocalAppData%\EnvVar\Logs\
```

---

### 4.2 文件命名

按天滚动：

```plaintext
YYYY-MM-DD.log
```

超过单文件大小限制后，继续使用同一天的递增后缀文件：

```plaintext
YYYY-MM-DD.1.log
YYYY-MM-DD.2.log
```

示例：

```plaintext
2026-04-12.log
```

---

### 4.3 滚动策略（Rolling）

* 按天生成日志文件
* 单文件最大大小：5 MB（可配置，默认开启）
* 超出后自动创建新文件

---

### 4.4 保留策略（Retention）

* 默认保留最近 **14 天**
* 超过时间自动删除

---

## 5. 日志内容规范

### 5.1 基本格式

每条日志必须包含：

* 时间（精确到秒）
* 日志级别
* 事件描述（Message）

---

### 5.2 上下文信息（必须支持）

对于关键操作和错误，必须记录上下文信息，例如：

```plaintext
Time: 2026-04-12 10:23:11
Level: Error
Action: Save Environment Variable
Variable: JAVA_HOME
Value: C:\Java\jdk17
Message: Access denied
```

---

### 5.3 必须记录的事件

#### 应用生命周期

* 应用启动（Application Started）
* 应用退出（Application Exiting）

---

#### 关键业务操作

* 添加环境变量
* 修改环境变量
* 删除环境变量
* 保存操作（成功 / 失败）
* 导入 / 导出

---

#### 异常（必须）

所有未捕获异常必须记录：

包括：

* UI 线程异常
* 非 UI 线程异常
* Task 未捕获异常

---

## 6. 异常捕获（必须实现）

需要全局捕获以下异常入口：

### 6.1 UI 线程

```csharp
Application.Current.DispatcherUnhandledException
```

---

### 6.2 非 UI 线程

```csharp
AppDomain.CurrentDomain.UnhandledException
```

---

### 6.3 Task 异常

```csharp
TaskScheduler.UnobservedTaskException
```

---

## 7. 用户设置（Settings）

在 Settings 中新增模块：

```plaintext
Logging
```

当前实现保留原有的 `Max History` 设置入口，并在其下新增独立的 `Logging` 入口。

---

### 7.1 配置项

#### 日志级别

```plaintext
Log Level:
- Error
- Warning
- Information（默认）
- Debug（可选）
```

---

#### 日志保留时间

```plaintext
Retention:
- 7 days
- 14 days（默认）
- 30 days
- 180 days
- Unlimited
```

---

#### 单文件最大大小

```plaintext
Max File Size:
- 默认 5 MB
- 当前实现中可在 Settings 中按 MB 配置为正整数
```

---

## 8. 验收标准

实现完成后应满足：

* 应用运行过程中产生日志文件
* 异常能够完整记录
* 日志文件自动按天生成
* 超过保留期自动删除
* 日志内容可读且包含上下文信息
